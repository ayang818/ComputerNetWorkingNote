# RDT3.0

如果信道即可能发生错误(在RDT3.0前默认是分组都一定到达)，也可能发生分组丢失怎么办

- checksum + serialize + ACK + 重传 够用吗


方法：发送方等待合理的时间
- 如果没有收到ACK，重传
- 如果分组或ACK只是延迟而不是丢了
    - 重传会产生重复，序列号机制可以处理
    - 接收方需要在ACK中显示告知所确认的分组
- 需要定时器
